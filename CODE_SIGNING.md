# Code Signing Policy

This document defines the code signing policy for official UniLoader Windows
releases.

Free code signing provided by [SignPath.io](https://about.signpath.io/), certificate by [SignPath Foundation](https://signpath.org/).

## Scope

This policy applies only to Windows installers and executables published in
the official [UniLoader GitHub Releases](https://github.com/Chucksterboy/UniLoader/releases).
Development builds, forks, locally compiled binaries, and third-party
repackages are outside this policy.

Until SignPath onboarding is complete, release artifacts may be unsigned.
Unsigned artifacts must not be represented as SignPath-signed.

## Source and Build Requirements

An official signed release must:

1. Be built from a version tag in the public
   [Chucksterboy/UniLoader](https://github.com/Chucksterboy/UniLoader)
   repository.
2. Use the committed dependency lockfiles and the public GitHub Actions release
   workflow.
3. Pass the frontend type check, backend test suite, and Rust Clippy checks
   configured in that workflow.
4. Produce SHA-256 checksum files alongside the Windows installers.
5. Be reviewed against the release tag and manually approved for signing before
   publication.

The signing certificate's private key is managed by SignPath and is never
stored in this repository or provided to project contributors.

## Roles and Approval

UniLoader is currently maintained as a single-maintainer project:

- **Author:** the contributor who prepares a change or release.
- **Reviewer:** the repository maintainer, Chucksterboy, who reviews external
  changes and verifies the release tag and automated checks.
- **Approver:** the repository maintainer, Chucksterboy, who manually approves
  an eligible artifact for signing.

The maintainer must not approve an artifact when its source, build provenance,
automated checks, dependency state, or release contents cannot be verified.
GitHub commit history and Actions logs form the public audit trail.

## Eligible Content

Only software whose source is available under the repository's
[MIT License](LICENSE), together with compatible open-source dependencies, is
eligible for signing. Proprietary payloads, bundled third-party mods, game
files, credentials, and user-generated profile bundles are not eligible
release contents.

Every bundled image, icon, and audio file must be listed in
[Bundled Asset Licenses](ASSET_LICENSES.md) with its provenance and license.
Assets with unknown provenance or licenses incompatible with this policy must
be removed before an artifact is approved for signing.

Remote game artwork and provider content fetched while UniLoader is running
are not part of the signed installer and retain their respective owners'
rights.

## Release and Verification

Official download pages must identify the release tag, link to this code
signing policy, and provide the SignPath attribution shown above. Release
artifacts are accompanied by SHA-256 checksum files generated by the public
release workflow.

Users should download UniLoader only from the official GitHub Releases page
and verify the publisher signature and checksum before installation.

## Incident Response

If a signed release is suspected of being compromised, malicious, built from
the wrong source, or signed contrary to this policy, the maintainer will:

1. Stop approving new signing requests.
2. Remove or clearly mark affected downloads.
3. Publish a security notice describing affected versions and remediation.
4. Contact SignPath to revoke affected artifacts or certificates when
   appropriate.
5. Correct the issue and require a new reviewed build before signing resumes.

Security concerns should be reported privately through
[GitHub Security Advisories](https://github.com/Chucksterboy/UniLoader/security/advisories/new).

## Privacy

UniLoader's handling of local data, credentials, profile exports, and network
requests is documented in the [Privacy Policy](PRIVACY.md).
